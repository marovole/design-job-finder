# Design Project Finder 🎨

一个 Claude Code 技能，用于自动搜索海外设计工作平台（Fiverr、Upwork、Dribbble等），找到正在寻找设计师的项目，提取客户联系方式，并生成个性化营销邮件来推广你的设计订阅服务。

## 功能特点

### 🔍 智能搜索
- **12个海外设计平台**: Fiverr, Upwork, Dribbble Jobs, 99designs, Behance, Toptal, LinkedIn, AngelList等
- **并行搜索架构**: 3批次并行，20-30分钟完成
- **深度信息提取**: 使用 Exa AI 深度研究引擎

### 📊 数据处理
- **智能去重**: 基于客户名+项目标题+平台的标准化算法
- **优先级评分**: 0-100分评分系统，A/B/C/D四级分类
- **联系方式提取**: 邮箱、LinkedIn、公司网站
- **预算解析**: 处理各种格式，统一转换为USD数值

### 📧 营销自动化
- **个性化邮件生成**: 基于客户类型和需求的AI邮件生成
- **语气自适应**: 专业正式/友好亲切/创意活泼
- **主题行推荐**: 每个项目生成3个备选主题行
- **批量处理**: 自动为A/B级项目生成邮件

### 📈 详细报告
- **CSV数据表**: 25+列完整项目信息
- **统计分析报告**: 多维度统计和TOP 10推荐
- **营销行动计划**: 预期转化率和发送策略
- **联系人列表**: 精简版联系方式清单

## 快速开始

### 前置要求

1. **Claude Code CLI** 已安装
2. **Exa AI MCP 服务器** 已配置（在 `~/.claude/.mcp.json` 中）
3. **Python 3.7+** 用于运行数据处理脚本

### 5分钟快速上手

1. **在 Claude Code 中激活技能**

在任何项目目录中，与 Claude 对话：

```
用户: 帮我找一些 Fiverr 和 Upwork 上的 UI/UX 设计项目，我想推广我的设计订阅服务 designsub.studio
```

Claude 会自动激活 `design-project-finder` 技能。

2. **回答配置问题**

技能会询问你：
- 目标设计类型（UI/UX、品牌、网页等）
- 目标平台（默认全选）
- 预算范围（可选）
- 客户类型偏好（可选）
- 邮件语气（专业正式/友好亲切/创意活泼/自适应）

3. **等待搜索完成**

技能会并行搜索多个平台，大约需要 20-30 分钟。你会看到进度更新。

4. **查看输出**

所有文件将保存在 `output/` 目录：
- `design_projects_YYYY-MM-DD.csv` - 完整项目数据
- `contact_list.csv` - 联系方式清单
- `design_projects_summary.md` - 统计报告
- `marketing_emails/` - 个性化邮件文件夹

## 详细使用流程

### 步骤 1: 需求确认

技能会询问以下信息：

**1. 目标设计类型**（多选）
- UI/UX 设计
- 品牌设计/Logo
- 网页设计/落地页
- 移动应用设计
- 插画设计
- 其他

**推荐**: 选择你最擅长的2-3种类型

**2. 目标平台**（默认全选）

主流平台：
- Fiverr - 全球最大自由职业平台
- Upwork - 高质量客户，长期项目为主
- Freelancer
- Guru

设计专业平台：
- Dribbble Jobs - 设计师专业社区
- Behance Jobs - Adobe 设计平台
- 99designs - 设计竞标平台
- Designhill

扩展渠道：
- Toptal - 高端设计师平台
- LinkedIn Jobs - 专业社交网络
- AngelList - 初创公司职位
- We Work Remotely - 远程工作平台

**推荐**: 首次使用选择 Fiverr + Upwork + Dribbble（3个主要平台）

**3. 项目预算范围**（可选）
- 小型项目 ($100-$500)
- 中型项目 ($500-$2,000)
- 大型项目 ($2,000+)
- 不限

**推荐**: 选择"中型项目"或"大型项目"以获得更高质量客户

**4. 客户类型偏好**（可选）
- 初创公司
- 中小企业
- 大型企业
- 个人创业者
- 不限

**5. 营销邮件语气**
- 专业正式 - 适合大企业和正式场合
- 友好亲切 - 适合初创公司和中小企业
- 创意活泼 - 适合个人创业者
- 自适应 - 根据客户类型自动选择（推荐）

### 步骤 2: 数据收集

技能会自动：

1. **启动深度研究任务**
   - 第一批: Fiverr, Upwork, Freelancer, Guru（4个平台并行）
   - 第二批: Dribbble Jobs, Behance, 99designs, Designhill（4个平台）
   - 第三批: Toptal, LinkedIn, AngelList, We Work Remotely（4个平台）

2. **提取项目信息**
   - 项目标题和详细要求
   - 预算范围
   - 发布时间和状态
   - 客户名称和类型
   - 客户行业和信誉

3. **提取联系方式**（重点）
   - 客户邮箱地址
   - LinkedIn 个人主页
   - 公司网站
   - 平台项目链接

**预计时间**: 20-30分钟
**预计结果**: 80-150个项目

### 步骤 3: 数据处理

技能会自动运行 `process_design_projects.py` 脚本：

1. **数据清洗**
   - 标准化预算格式（统一为USD）
   - 解析发布时间
   - 验证邮箱格式
   - 提取行业标签

2. **去重处理**
   - 标准化公司名称
   - 提取项目关键词
   - 比较联系信息完整度
   - 保留信息更完整的版本

3. **优先级评分**
   - 预算权重（40分）
   - 联系方式完整度（30分）
   - 项目紧急度（15分）
   - 客户质量（15分）

4. **分级标签**
   - A级-极高优先（70-100分）
   - B级-高优先（50-69分）
   - C级-中优先（30-49分）
   - D级-低优先（0-29分）

### 步骤 4: 邮件生成

技能会自动运行 `generate_marketing_emails.py` 脚本：

1. **筛选项目**
   - 默认只为 A级 和 B级 项目生成邮件
   - 大约 40-60 封邮件

2. **选择语气**
   - 根据客户类型自动适配
   - 或使用你指定的语气

3. **生成邮件**
   - 调用 LLM 生成个性化正文
   - 生成 3 个备选主题行
   - 包含项目摘要信息

4. **保存文件**
   - 高优先级: `marketing_emails/high_priority/`
   - 中优先级: `marketing_emails/medium_priority/`
   - 文件名: `project_XXX_[客户名]_email.md`

### 步骤 5: 查看输出

所有文件在 `output/` 目录中：

```
output/
├── design_projects_2026-01-08.csv      # 主数据文件
├── contact_list.csv                    # 联系方式清单
├── design_projects_summary.md          # 统计报告
├── marketing_emails/                   # 邮件文件夹
│   ├── high_priority/                  # A/B级项目
│   │   ├── project_001_TechStartup_email.md
│   │   ├── project_002_DesignCo_email.md
│   │   └── ...
│   └── medium_priority/                # C级项目（可选）
├── email_batch_template.md             # 批量发送指南
└── README.md                           # 使用说明
```

## 常见问题

### Q1: 搜索需要多长时间？

**A**: 通常 20-30 分钟。

- 单平台: 2-3分钟
- 3个平台: 10-15分钟
- 12个平台: 20-30分钟

使用并行搜索大大减少了总时间。

### Q2: 能找到多少个项目？

**A**: 通常 80-150 个项目（12个平台）。

- 每个平台: 6-15个项目
- 去重后: 减少10-15%
- 有效联系方式: 60-70%

### Q3: 联系方式提取率有多高？

**A**: 平均 60-70%。

不同平台差异较大：
- LinkedIn, Toptal: 90-100%（平台特性）
- Upwork, Dribbble: 80-90%
- Fiverr, 99designs: 60-70%
- 其他平台: 50-60%

**提升技巧**:
- 使用 `exa-research-pro` 模型
- 在 prompt 中强调联系方式提取
- 补充公司网站搜索

### Q4: 邮件会不会太模板化？

**A**: 不会，如果正确使用。

**自动生成的邮件**:
- 基于项目具体需求个性化
- 根据客户类型调整语气
- 自然融入 designsub.studio

**最佳实践**:
- 人工审核前5-10封邮件
- 根据反馈调整 prompt
- 添加个人化细节

### Q5: 如何避免邮件被标记为垃圾邮件？

**A**: 遵循以下最佳实践：

1. **个性化**
   - 提到具体项目需求
   - 使用客户真实姓名
   - 不使用模板化语言

2. **技术设置**
   - 使用正规商业邮箱（有SPF/DKIM）
   - 小批量发送（每天10-15封）
   - 避免太多链接或附件

3. **内容质量**
   - 避免全大写、过多感叹号
   - 提供真实联系信息
   - 包含退订选项（如适用）

### Q6: 如何跟踪营销效果？

**A**: 推荐工具和方法：

**邮件跟踪**:
- Mailtrack（免费Gmail插件）
- HubSpot（免费CRM+邮件跟踪）
- Streak（Gmail CRM）

**CRM系统**:
- Airtable（灵活且免费）
- Notion（适合个人使用）
- HubSpot（功能全面）
- Pipedrive（专业销售CRM）

**关键指标**:
- 发送数量
- 打开率（目标25-35%）
- 响应率（目标8-12%）
- 会议预约数
- 转化为客户数

### Q7: 多久应该重新运行一次？

**A**: 推荐 **每2周** 运行一次。

原因：
- 新项目持续发布
- 避免联系已处理过的项目
- 保持客户线索新鲜度

### Q8: 可以用于其他设计订阅服务吗？

**A**: 当然可以！

修改邮件生成脚本中的服务介绍：

```python
# 在 generate_email_prompt() 中修改
**About [你的服务名称]**:
We are a design subscription service offering:
- [你的特色1]
- [你的特色2]
- ...
```

### Q9: 某个平台搜索失败怎么办？

**A**: 几种解决方案：

1. **重试该平台**
   - Exa AI 有时会遇到临时问题
   - 等待几分钟后重试

2. **调整 prompt**
   - 更具体描述平台URL
   - 简化搜索条件

3. **使用备用平台**
   - 12个平台中失败1-2个是正常的
   - 专注于成功的平台

4. **手动补充**
   - 直接访问平台搜索
   - 手动添加到CSV

### Q10: 如何定制邮件风格？

**A**: 三种方法：

1. **使用模板库**
   - 参考 `templates/email_templates.md`
   - 复制喜欢的风格

2. **修改 prompt**
   - 编辑 `generate_email_prompt()` 函数
   - 调整语气、长度、结构要求

3. **人工后期编辑**
   - 让AI生成初稿
   - 人工添加个性化细节

## 最佳实践

### 🎯 搜索策略

**首次使用**:
- 选择 2-3 种你最擅长的设计类型
- 选择 3-5 个主要平台（避免信息过载）
- 设定预算下限（$500+）以筛选优质客户

**定期使用**:
- 每2周运行一次
- 逐步扩展到更多平台
- 根据转化率优化平台选择

### 📧 邮件发送策略

**第1周 - 测试阶段**:
- 发送 5-10 封邮件（最高分的 A级项目）
- 人工仔细审核每封邮件
- 跟踪打开率和回复率

**第2周 - 优化阶段**:
- 根据反馈调整邮件模板
- 增加到每天 10-15 封
- A/B 测试不同主题行

**第3周+ - 规模化阶段**:
- 每天 15-20 封
- 使用 CRM 自动化跟进
- 持续优化转化率

### 📊 数据管理

**导入 CRM**:
1. 导出 `contact_list.csv`
2. 导入 Airtable 或 HubSpot
3. 添加自定义字段:
   - 联系状态（未联系/已发送/已回复/已成交）
   - 回复日期
   - 下次跟进日期
   - 备注

**持续优化**:
- 记录每个平台的转化率
- 标记最有效的客户类型
- 调整搜索参数

### 🔄 跟进流程

**时间线**:
- Day 0: 首次发送
- Day 2-3: 第一次跟进（未回复）
- Day 7-10: 第二次跟进（最后一次）
- Day 14+: 标记为"不感兴趣"，移除

**跟进模板**:
参考 `templates/email_templates.md` 中的跟进邮件部分

## 技术架构

### 核心组件

1. **SKILL.md** - 技能主文件
   - 工作流程定义
   - Prompt 模板
   - 数据结构规范

2. **process_design_projects.py** - 数据处理引擎
   - 去重算法
   - 优先级评分
   - CSV生成
   - 统计报告

3. **generate_marketing_emails.py** - 邮件生成器
   - 语气选择逻辑
   - LLM prompt 构建
   - 文件组织

4. **templates/email_templates.md** - 模板库
   - 多种语气示例
   - 客户类型案例
   - 话术库

### 依赖项

**Claude Code**:
- Skill 系统
- MCP 集成

**Exa AI MCP**:
- `deep_researcher_start` - 启动深度搜索
- `deep_researcher_check` - 轮询结果

**Python 3.7+**:
- 标准库（csv, re, datetime, typing）
- 无需额外依赖

### 数据流

```
用户输入
    ↓
需求确认（设计类型、平台、预算等）
    ↓
并行深度搜索（12个平台 × Exa AI）
    ↓
原始数据提取（项目+联系方式）
    ↓
数据清洗 & 去重（process_design_projects.py）
    ↓
优先级评分 & 分级
    ↓
邮件生成（generate_marketing_emails.py）
    ↓
输出文件（CSV + MD + 邮件文件）
    ↓
用户使用（发送邮件、跟踪转化）
```
